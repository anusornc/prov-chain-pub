These exports support R002 bottleneck analysis only. They are not primary paper
comparison evidence because they are ProvChain-only profiling campaigns. The
`walsync100` export is relaxed-durability evidence because WAL and chain-index
fsync intervals were set to `100`.
The `batchblock` export is also relaxed-durability diagnostic evidence and uses
different write semantics: `100` RDF triples are admitted as one block rather
than `100` separate blockchain transactions.
The `coldsteady` export separates cold dataset load from post-load append under
conservative WAL/index sync settings; it is phase-separation evidence, not a
cross-system ledger/write comparison.

Scope note:

- the trace-query export supports only the `ProvChain vs Neo4j` trace-query/provenance baseline
- the Fluree trace-query export supports only the `ProvChain vs Neo4j vs Fluree` trace-query/provenance evidence campaign
- the semantic export supports only `ProvChain vs Fluree` semantic admission evidence with native/externalized capability boundaries
- the ledger-write export supports only the `ProvChain vs Fabric` ledger/write-path evidence for `B013`
- the Geth ledger-write export supports only `ProvChain vs Geth` public-chain-baseline evidence for `B017`
- the reference policy export supports only historical `Fabric` governance/policy evidence for `B014`
- the reference ProvChain profiling export supports only R002 write-path bottleneck analysis
- the final R002 bulk dataset-admission export supports only the import-row algorithm remediation claim, not per-transaction ledger/write or finality claims
- the policy workload pack export supports only historical Fabric governance/policy scenario-pack evidence for `B019`
- the comparative governance-policy export supports `ProvChain vs Fabric` policy scenarios only with the `cross-model-with-caveat` fairness label for ProvChain rows; the current primary export includes both server-reported decision latency and client-observed policy API round-trip rows
- the scale-up reference export supports only trace-query ranking stability on `supply_chain_5000`; it remains reference/confidence evidence unless explicitly promoted by a later publication decision
- the Fabric ledger-write export does not support trace-query, semantic, or policy-family claims
- the Geth ledger-write export does not support permissioned-ledger parity, trace-query, semantic, or policy-family claims
- the ProvChain profiling export does not support cross-system ledger/write comparison claims
- scale-up input files do not support scaling claims by themselves; only the validated reference export listed above supports the current scale-up confidence claim
